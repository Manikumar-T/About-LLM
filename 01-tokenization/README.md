# Tokenization

## What is it?
Tokenization is the process of splitting raw text into smaller units called **tokens**. These tokens are the basic input unit for every language model.

## Why does it matter?
- Models don't understand raw text — they work with numbers.
- Each token is mapped to an integer ID that the model processes.
- The choice of tokenization strategy directly affects vocabulary size, model efficiency, and how well rare words are handled.

## Common strategies

| Strategy | Example input | Tokens |
|----------|--------------|--------|
| Word      | `"Hello world"` | `["Hello", "world"]` |
| Character | `"Hi"` | `["H", "i"]` |
| BPE (Byte-Pair Encoding) | `"unhappiness"` | `["un", "happiness"]` |
| WordPiece | `"tokenization"` | `["token", "##ization"]` |

Modern LLMs (GPT, LLaMA, etc.) use **BPE** or **SentencePiece**, which balance vocabulary size with coverage of rare words.

## Key concepts
- **Vocabulary**: the fixed set of all known tokens.
- **Token ID**: the integer index of a token in the vocabulary.
- **[UNK] token**: used when a token is not in the vocabulary.
- **Context window**: the maximum number of tokens a model can process at once.
