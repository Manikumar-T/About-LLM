# Transformer

## What is it?
The **Transformer** is the neural network architecture behind all modern LLMs (GPT, BERT, LLaMA, etc.). Introduced in the 2017 paper *"Attention is All You Need"* by Vaswani et al.

## High-level architecture

```
Input tokens
    ↓
Token Embeddings + Positional Encoding
    ↓
┌─────────────────────────────┐
│  Encoder / Decoder Block    │  × N layers
│  ┌─────────────────────┐   │
│  │  Multi-Head Attention│   │
│  └─────────┬───────────┘   │
│        Add & Norm           │
│  ┌─────────────────────┐   │
│  │  Feed-Forward (FFN) │   │
│  └─────────┬───────────┘   │
│        Add & Norm           │
└─────────────────────────────┘
    ↓
Output logits → softmax → next token
```

## Key components

| Component | Purpose |
|-----------|---------|
| **Positional Encoding** | Injects token order (sin/cos waves by position) |
| **Multi-Head Attention** | Attends to multiple relationship types in parallel |
| **Feed-Forward Network** | Per-token non-linear transformation (2 linear layers + ReLU) |
| **Residual Connection** | Adds input to output — stabilises deep networks |
| **Layer Norm** | Normalises activations — speeds up training |

## Positional Encoding formula
```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```
Different frequencies encode different positions without any learned parameters.

## Encoder vs Decoder
- **Encoder-only** (BERT): bidirectional, good for classification/understanding.
- **Decoder-only** (GPT, LLaMA): causal (masked) attention, good for generation.
- **Encoder-Decoder** (T5, original Transformer): best for seq-to-seq tasks (translation, summarisation).
