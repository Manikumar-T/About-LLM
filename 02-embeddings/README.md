# Embeddings

## What is it?
An **embedding** is a dense vector of numbers that represents a token, word, sentence, or document. Similar meanings produce similar vectors.

## Why does it matter?
- Models need a way to understand *relationships* between words numerically.
- Embeddings let the model know that "king" and "queen" are more related than "king" and "banana".
- They are the bridge between discrete tokens and continuous mathematics.

## How they work
Each token is looked up in an **embedding table** (a matrix of shape `vocab_size × embedding_dim`). The row for that token ID is its embedding vector — learned during training via backpropagation.

## Key properties
- **Semantic similarity**: cosine similarity between vectors reflects meaning similarity.
- **Arithmetic**: famous example — `king − man + woman ≈ queen`.
- **Dimensionality**: typical LLMs use 768–8192 dimensions per token.

## Cosine Similarity
```
similarity = (A · B) / (|A| × |B|)   range: -1 (opposite) to 1 (identical)
```
