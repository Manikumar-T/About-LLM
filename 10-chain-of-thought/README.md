# Chain-of-Thought (CoT)

## What is it?
**Chain-of-Thought prompting** encourages an LLM to produce intermediate reasoning steps before giving a final answer. This dramatically improves accuracy on tasks that require logic, math, or multi-step reasoning.

## Why does it matter?
- LLMs are next-token predictors. Without CoT, they jump straight to an answer and can get multi-step problems wrong.
- CoT forces the model to "show its work", making each step more likely to be correct.
- Emergent behaviour: CoT only works well in models with ≥ ~7B parameters.

## Variants

### Zero-shot CoT
Just add a trigger phrase — no examples needed.
```
Question: ...
Let's think step by step:
```

### Few-shot CoT
Provide 2–5 worked examples with full reasoning chains before the actual question.
```
Question: Sarah has 5 apples...
Step 1: Start with 5
Step 2: Add 3 → 8
Step 3: Subtract 2 → 6
Answer: 6

Question: <real question>
Let's think step by step:
```

### Self-consistency
Generate multiple CoT reasoning paths (with temperature > 0), then take the majority vote answer. More reliable than a single path.

### Tree-of-Thoughts (ToT)
Explore multiple reasoning branches at each step, backtrack dead ends. Better for search and planning problems.

### ReAct (Reason + Act)
Interleave reasoning steps with tool calls (search, calculator, API). The model reasons, acts, observes, then reasons again.

## When to use CoT

| Task | CoT helps? |
|------|-----------|
| Arithmetic / math | Yes — large gains |
| Multi-step logic | Yes |
| Commonsense reasoning | Yes |
| Simple factual Q&A | Rarely — adds unnecessary tokens |
| Classification | Rarely |

## Key insight
CoT works because the model's **intermediate tokens become a scratchpad**. Each generated token conditions the next, so reasoning in writing lets the model build up a correct answer incrementally.
