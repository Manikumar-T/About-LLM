# Model Context Protocol (MCP)

## What is it?
**MCP** is an open standard (by Anthropic, 2024) that defines how LLM applications connect to external data sources and tools via a unified client-server protocol. Think of it as USB-C for AI integrations вҖ” one protocol, any tool.

## Why does it matter?
Without MCP, every AI app hand-rolls its own integrations. MCP standardises the interface so any MCP-compatible host (Claude Desktop, VS Code, custom agents) can use any MCP server (GitHub, databases, filesystems, APIs) without custom glue code.

## Three capability types

### Tools
Callable functions the model can invoke (equivalent to function calling).
```json
{ "name": "get_weather", "description": "...", "inputSchema": { ... } }
```

### Resources
Read-only data the model can access вҖ” files, database rows, live API data.
```
URI: file:///docs/guide.txt
URI: postgres://db/users/42
URI: https://api.example.com/data
```

### Prompts
Reusable, parameterised prompt templates stored on the server.
```json
{ "name": "summarise", "arguments": [{ "name": "text" }, { "name": "style" }] }
```

## Protocol: JSON-RPC 2.0

All messages follow JSON-RPC 2.0 format:

**Request:**
```json
{ "jsonrpc": "2.0", "id": 1, "method": "tools/list", "params": {} }
```

**Response:**
```json
{ "jsonrpc": "2.0", "id": 1, "result": { "tools": [ ... ] } }
```

## Key methods

| Method | Direction | Purpose |
|--------|-----------|---------|
| `initialize` | client вҶ’ server | Handshake, exchange capabilities |
| `tools/list` | client вҶ’ server | Discover available tools |
| `tools/call` | client вҶ’ server | Execute a tool |
| `resources/list` | client вҶ’ server | Discover available resources |
| `resources/read` | client вҶ’ server | Read a resource |
| `prompts/list` | client вҶ’ server | Discover prompt templates |
| `prompts/get` | client вҶ’ server | Render a prompt with arguments |

## Transport options

| Transport | Use case |
|-----------|---------|
| **stdio** | Local servers launched as subprocesses (most common) |
| **HTTP + SSE** | Remote servers, server-sent events for streaming |
| **WebSocket** | Bidirectional streaming |

## MCP vs plain Tool Calling

| | Tool Calling | MCP |
|-|-------------|-----|
| Scope | Single LLM request | Persistent server session |
