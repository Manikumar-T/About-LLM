# Attention Mechanism

## What is it?
The **attention mechanism** lets the model decide which tokens in the input are most relevant when producing each output token. It is the core innovation of the Transformer architecture.

## Scaled Dot-Product Attention

```
Attention(Q, K, V) = softmax( Q·Kᵀ / √d_k ) · V
```

| Symbol | Meaning |
|--------|---------|
| Q | Query — "what am I looking for?" |
| K | Key   — "what do I contain?" |
| V | Value — "what do I return if selected?" |
| d_k | Dimension of keys (scaling factor) |

## Step-by-step
1. Compute a **score** between the query and every key (dot product).
2. **Scale** by √d_k to prevent vanishing gradients in softmax.
3. Apply **softmax** to get attention weights (sum to 1).
4. Take a **weighted sum** of the values using those weights.

## Why scaling?
Without dividing by √d_k, dot products grow large in high dimensions, pushing softmax into saturation (near-zero gradients).

## Self-attention vs Cross-attention
- **Self-attention**: Q, K, V all come from the same sequence (the model attends to itself).
- **Cross-attention**: Q comes from the decoder, K/V from the encoder (used in encoder-decoder models).

## Multi-Head Attention
Real transformers run attention `h` times in parallel with different learned projections, then concatenate — letting each head specialize in different relationships.
