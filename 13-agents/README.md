# AI Agents

## What is it?
An **AI agent** is an LLM that autonomously decides what actions to take, executes them via tools, observes the results, and repeats — until it reaches a goal. Unlike a single prompt-response, an agent operates in a loop.

## Why does it matter?
- A single LLM call can answer a question. An agent can **complete a multi-step task**.
- Agents can browse the web, write and run code, manage files, call APIs, and chain results together — all without human intervention at each step.
- They are the foundation of products like Cursor, Devin, and Claude Code.

## The ReAct Loop (Reason + Act)

```
┌─────────────────────────────────────────────┐
│                  AGENT LOOP                 │
│                                             │
│   ┌──────────┐   Thought (why + what)       │
│   │   LLM    │──────────────────────────▶  │
│   │ (Planner)│                  ┌────────┐  │
│   └──────────┘◀─── Observation─│  Tool  │  │
│        ▲            (result)    └────────┘  │
│        │                            ▲       │
│        └──── Memory (full history) ─┘       │
└─────────────────────────────────────────────┘
              ↓ when done
         Final Answer
```

Each iteration has three phases:

| Phase | What happens |
|-------|-------------|
| **Thought** | LLM reasons about what to do next (scratchpad) |
| **Action** | LLM picks a tool and provides arguments |
| **Observation** | Tool runs, result is fed back into context |

## Example trace

```
Task: "How far does light travel in 10 seconds?"

Step 1
  Thought    : I'll look up the speed of light.
  Action     : search({"query": "speed of light"})
  Observation: {"result": "299,792,458 metres per second"}

Step 2
  Thought    : Now I'll calculate distance = speed × time.
  Action     : calculator({"expression": "299792458 * 10"})
  Observation: {"result": 2997924580}

Step 3
  Thought    : Calculation done.
  Action     : finish({"answer": "Light travels 2,997,924,580 metres in 10 seconds."})

Final Answer: Light travels 2,997,924,580 metres in 10 seconds.
```

## Agent memory types

| Memory type | Description |
|-------------|-------------|
| **In-context** | Full conversation history kept in the prompt (limited by context window) |
| **External / long-term** | Vector DB or key-value store persisted across sessions |
| **Episodic** | Log of past actions and outcomes the agent can query |
| **Scratchpad** | Working memory for the current task — thoughts and observations |

## Agent patterns

### Single agent
One LLM in a ReAct loop with tools. Simple, predictable.

### Multi-agent (orchestrator + subagents)
One orchestrator LLM breaks a task into subtasks and delegates to specialist agents (researcher, coder, reviewer). Results are aggregated.

```
Orchestrator
  ├── Research Agent  → web search, RAG
  ├── Coder Agent     → write & run code
  └── Critic Agent    → review output
```

### Reflection / self-critique
After producing an answer, the agent critiques its own output and improves it before returning — no human in the loop.

## Key design considerations

| Concern | Approach |
|---------|---------|
| **Infinite loops** | Set a max-steps limit |
| **Hallucinated tool calls** | Validate arguments strictly before execution |
| **Cost** | Each step = one LLM call; keep loops short |
| **Safety** | Require confirmation before irreversible actions (delete, send email) |
| **Observability** | Log every thought/action/observation for debugging |

## Difference from plain tool calling

| | Tool Calling | Agent |
|-|-------------|-------|
| Iterations | 1 | Many (loop) |
| Planning | No | Yes (multi-step) |
| Goal-directed | No | Yes |
| Memory across steps | No | Yes |

## Popular frameworks
- **LangChain / LangGraph** — graph-based agent orchestration
- **AutoGen** — multi-agent conversation framework (Microsoft)
- **CrewAI** — role-based multi-agent teams
- **Claude Code** — coding agent built on Anthropic's API
- **Anthropic Agent SDK** — low-level primitives for building agents
