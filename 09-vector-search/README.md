# Vector Search

## What is it?
**Vector search** (a.k.a. nearest-neighbour search) finds the stored vectors that are most similar to a query vector. It is the retrieval backbone of RAG systems, semantic search, and recommendation engines.

## Why does it matter?
- Text, images, and audio can all be encoded as vectors by neural networks.
- Finding semantically similar content becomes a pure math problem: find nearby vectors.
- Powers every modern search engine upgrade and AI assistant with memory.

## Similarity metrics

| Metric | Formula | Best for |
|--------|---------|---------|
| **Cosine similarity** | `(A·B) / (|A||B|)` | Text embeddings (direction matters) |
| **Euclidean distance** | `√Σ(aᵢ−bᵢ)²` | Dense numeric features |
| **Dot product** | `A·B` | When vectors are already normalised |

## Index types

### Flat (Brute-force)
- Compare query to every stored vector.
- **Exact** results, **O(n)** per query.
- Fine for < 100k vectors.

### IVF (Inverted File Index)
- Cluster vectors into buckets; search only nearby buckets.
- Approximate, much faster for millions of vectors.

### HNSW (Hierarchical Navigable Small World)
- Graph-based index. Navigate from coarse to fine layers.
- State-of-the-art speed/recall trade-off. Used by Chroma, Weaviate, pgvector.

### Quantisation (PQ / SQ)
- Compress vectors to fewer bits to reduce memory and increase speed.
- Slight accuracy loss, 4–64× compression.

## Popular vector databases
| Database | Notes |
|----------|-------|
| **FAISS** | Meta's library, CPU/GPU, production-grade |
| **Chroma** | Python-native, easy to use locally |
| **Pinecone** | Managed cloud service |
| **Weaviate** | Open-source, multi-modal |
| **pgvector** | Vector search inside PostgreSQL |

## Typical RAG pipeline
```
Document → chunk → embed → store in vector DB
Query    →         embed → search vector DB → top-k chunks → LLM
```
