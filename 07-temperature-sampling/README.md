# Temperature & Sampling

## What is it?
After a Transformer computes **logits** (raw scores for each token in the vocabulary), a **sampling strategy** decides which token to actually output. Temperature and sampling methods control the trade-off between creativity and determinism.

## Temperature

```
scaled_logit = logit / temperature
probabilities = softmax(scaled_logits)
```

| Temperature | Effect |
|-------------|--------|
| → 0 (greedy) | Always picks the highest-probability token. Deterministic, repetitive. |
| 0.1 – 0.5   | Conservative, focused, good for factual tasks. |
| 1.0          | Raw model distribution. Balanced. |
| > 1.0        | Flatter distribution, more random and creative. |

## Sampling strategies

### Greedy decoding
Always pick `argmax(logits)`. Fast but prone to repetition.

### Random sampling
Sample from the full softmax distribution. Can produce unlikely tokens.

### Top-k sampling
Keep only the **k** highest-probability tokens, re-normalise, then sample.
- Prevents very unlikely tokens from being chosen.
- `k=50` is a common default.

### Top-p (Nucleus) sampling
Keep the **smallest set** of tokens whose cumulative probability ≥ p, then sample.
- Adapts to the shape of the distribution dynamically.
- `p=0.9` or `p=0.95` are common defaults.

### Beam search
Maintain the **B** most likely partial sequences at each step. Deterministic. Good for translation and summarisation, less good for open-ended generation.

## Typical settings by task

| Task | Temperature | Sampling |
|------|-------------|---------|
| Code generation | 0.2 | Greedy / top-k |
| Factual Q&A | 0.3–0.5 | Top-p 0.9 |
| Creative writing | 0.8–1.2 | Top-p 0.95 |
| Brainstorming | 1.0–1.5 | Top-k 50 |
